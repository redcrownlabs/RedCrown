# Security Policy

## Supported versions

RedCrown has not published a supported release yet. Security fixes currently
target the `main` branch.

## Reporting a vulnerability

Do not open a public issue for a suspected vulnerability. Use GitHub's private
security-advisory reporting for this repository and include:

- the affected commit or component;
- reproduction steps or a minimal proof of concept;
- the expected impact;
- any known mitigations.

Do not include real credentials, private catalog data, watch history, or
copyrighted media in a report. If private vulnerability reporting is
temporarily unavailable, open a public issue that asks the maintainers to
enable a private contact channel without disclosing vulnerability details.

The maintainers will acknowledge a complete report when available, investigate
it privately, and coordinate disclosure after a fix is ready. No response-time
or bug-bounty commitment is made while the project is pre-release.
